## Naming conventions going forward
- **`scripts/NN_description.py`** — numbered, run in that order, thin (argparse + a couple of calls into `src/`)
- **`src/<area>/<thing>.py`** — reusable logic, no CLI, importable, unit-testable
- **`tests/test_<thing>.py`** — mirrors the `src/` module it tests
- **`configs/<dataset>_<compression>.yaml`** — one config file per dataset/setting combo

## Changelog
- **Fixed** (`src/data/ffpp_splits.py`): fake-video resolution previously tried
  `a_b.mp4` then `b_a.mp4` and stopped at the first match, silently dropping
  ~50% of fake videos per split. Confirmed via the official downloader
  (`external/faceforensics_download_v4.py`) that BOTH orderings are separate,
  real files on the server for every pair. Now resolves both as distinct
  `VideoItem`s. Synthetic test updated accordingly (expects 2x fake count).
- **Note**: `timm.create_model("xception", ...)` internally maps to
  `legacy_xception` with a deprecation warning — harmless, same architecture.
- **Design note** (`src/models/fusion_model.py`): CLIP contributes a pooled
  global feature (not spatial tokens) to the fusion; the localization head
  runs on the frequency branch's spatial map instead, which keeps full
  resolution throughout. This was a deliberate simplification over extracting
  CLIP patch tokens (which varies across open_clip versions/model configs) —
  documented in the module docstring. Revisit if patch-token-level fusion
  turns out to matter empirically.
- **requirements.txt**: `open_clip_torch` pinned to `3.3.0` (the version
  actually tested here) rather than an earlier guess of `2.29.0`.
- **Feature added**: `--exclude_methods` on `scripts/04_train_baseline.py` and
  `scripts/06_train_fusion.py` for genuine cross-manipulation leave-one-out
  training. Required updating `build_dataloaders()` (`src/data/dataset.py`) to
  filter train/val by `methods` but always keep `test` unfiltered by default
  (new `test_methods="all"` param) — otherwise the held-out method wouldn't be
  present at test time to measure generalization to it.
